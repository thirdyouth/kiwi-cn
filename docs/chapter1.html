第一章：介绍¶

KIWI 是什么？¶

KIWI 是一个命令行工具，由 Perl 编写，用于构建 Linux 系统镜像，支持多种镜像格式。KIWI 也是 SUSE Sudio 的后台应用实例构建的后端工具，同时也在 OpenSUSE 构建服务 中用于构建镜像，其中包行所有 SUSE 产品镜像。

Linux 镜像可能会以多种格式呈现，比如 *.iso 文件可以下载后，将发行版安装文件烧录到外部媒介，那么它就是一个镜像。虚拟机系统，如 KVM，Xen，或者 VMware 所使用的文件也是一个镜像。安装在您的硬盘中的Linux系统可以通过 dd 命令导出到一个文件中，这个文件也是一个镜像。

通常来说，您可以认为一个镜像就是一个包含 Linux 系统的文件。根据文件类型的不同，您可以以不同的方式对镜像进行使用。例如，可以将 ISO 镜像烧录到外部媒介，然后从 CD/DVD 进行系统启动，或者可以在虚拟机中运行硬盘中的 *.iso 文件。KIWI 支持以下类型和格式的镜像：

  • ISO
  • Live CD/DVD
  • PXEBoot
  • 硬盘
  • USB
  • Amazon EC2(.ami)
  • Docker
  • Google Cloud Format(..gce)
  • KVM/Qemu(.qcow2)
  • 开发虚拟格式 (.ovf, .ova)
  • Vagrant(.vagrant)
  • VirtualBox(.vdi)
  • VMware(.vmdk)
  • XEN

KIWI 用来做什么？¶

KIWI 可以允许您以多种格式配置，构建，和部署您自己的操作系统镜像。KIWI的工作流分为两个明显的阶段。有关镜像创建生成的具体细节可以参考 第三章：基本工作流程 。

  1. 准备 。创建一个根目录，用于容纳新文件系统中的内容。从软件包源安装所需要的软件包，例如SUSE Linux 商业服务器的安装媒介，或者在线仓库。创建镜像描述文件（config.xml），和可选应用定制化内容。该操作将会生成 “未打包根目录树” 。
  2. 创建 。镜像本身使用上一步中生成的未打包根目录树进行创建。镜像创建过程不需要用户操作，但是可以通过修改 images.sh 脚本文件进行优化，该脚本在创建过程中会被调用执行。

KIWI 怎么使用？¶

KIWI 是一个命令行工具，可以在终端中输入 kiwi 进行调用。KIWI 需要以 root 权限进行运行，因为对于镜像创建过程中的许多操作都需要管理员权限才可以执行。所以，在使用 KIWI 时，您需要特别注意自己当前正在做的事情，以及关注它们的执行顺序。在您的系统上运行 KIWI 对您的系统本身不具有很大的危险，但是需要注意您正使用的是 root 用户。

如 KIWI 用来做什么？ 中所描述的，镜像的创建只需要两个步骤。第一步使用 kiwi --prepare ，第二步执行 kiwi --create 。为了方便起见，KIWI 可以使用 --build 将这两个步骤合起来运行。
